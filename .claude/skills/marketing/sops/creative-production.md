# SOP: Creative Production

## Purpose
Standard operating procedure for producing visual assets and creative content for marketing campaigns.

## Scope
- Ad creatives (images, videos)
- Social media graphics
- YouTube thumbnails
- Platform banners and covers

## Responsible Agents
- **Owner**: Pixel (Visual Graphic Artist)
- **Copy**: Chris (Chief Storyteller)
- **Video**: Spike (Remotion Video)
- **Requester**: Adler, Project Leads
- **Approver**: Drew (Project Manager)

---

## Creative Request Process

### 1. Request Submission

**Requester provides:**
- [ ] Platform(s) needed
- [ ] Asset type (ad image, thumbnail, banner, etc.)
- [ ] Dimensions required (or "see specs")
- [ ] Copy/text to include
- [ ] Brand (which brand guidelines apply)
- [ ] Due date
- [ ] Reference images (optional but helpful)
- [ ] Campaign context (what's the goal)

### 2. Request Intake

**Pixel Actions:**
1. Acknowledge request within 4 hours
2. Clarify any missing information
3. Confirm dimensions from platform specs
4. Estimate delivery time
5. Add to production queue

## Production Workflow

### Phase 1: Research & Concept (Day 1)

1. **Review Brand Guidelines**
   - Check brand system: `.claude/skills/pptx-generator/brands/[brand]/brand-system.md`
   - Confirm color palette
   - Verify font requirements
   - Note any do's/don'ts

2. **Review Platform Specs**
   - Confirm dimensions
   - Check safe zones
   - Note text limits
   - Reference: `.claude/skills/marketing/platforms/`

3. **Concept Development**
   - Apply Hyper-Dopamine principles:
     - Pattern interrupt potential
     - Intrigue/curiosity elements
     - Big specific benefit visualization
   - Sketch 2-3 concepts

### Phase 2: Production (Day 1-2)

1. **Create Draft Versions**
   - Produce 2-3 variations
   - Follow design principles
   - Apply brand guidelines
   - Test at actual display size

2. **Quality Checklist**
   - [ ] Correct dimensions
   - [ ] Within safe zones
   - [ ] Text readable on mobile
   - [ ] Brand colors accurate
   - [ ] High contrast/visibility
   - [ ] File size within limits
   - [ ] Correct file format

### Phase 3: Review (Day 2)

1. **Internal Review**
   - Self-check against brief
   - Mobile preview test
   - Brand compliance check

2. **Stakeholder Review**
   - Share with requester
   - Get feedback within 24 hours
   - Note revision requests

### Phase 4: Revision & Delivery (Day 3)

1. **Incorporate Feedback**
   - Make requested changes
   - Don't exceed 2 revision rounds
   - Escalate if feedback unclear

2. **Final Delivery**
   - Export in required formats
   - Follow naming convention
   - Upload to shared location
   - Notify requester

## Design Principles

### Hyper-Dopamine Approach (For Ads)

1. **Pattern Interrupt**
   - Unusual visuals that stop the scroll
   - Unexpected color combinations
   - Faces showing strong emotion
   - Something "off" that demands attention

2. **Dual Image Technique**
   - Large primary image
   - Smaller secondary image for context
   - Creates curiosity and intrigue
   - Forces viewer to process multiple elements

3. **Highlight Elements**
   - Red circles around key items
   - Arrows pointing to important details
   - Selective blur/focus
   - Before/after positioning

4. **Native Feel**
   - Avoid overly polished stock look
   - iPhone-quality photos often win
   - User-generated content aesthetic
   - Don't look like an ad

### Thumbnail Best Practices

1. **Faces**
   - Strong emotion (surprise, excitement, curiosity)
   - Eye contact with viewer
   - Large in frame

2. **Text**
   - 3-5 words maximum
   - Large, bold, readable
   - High contrast against background
   - Avoid platform-covered areas

3. **Colors**
   - Bright, saturated colors
   - Yellow, red, orange perform best
   - Strong contrast between elements
   - Avoid muted/pastel

4. **Composition**
   - Rule of thirds
   - Clear focal point
   - Minimal clutter
   - Breathing room around elements

## File Naming Convention

```
[platform]-[type]-[campaign]-[variant]-[date].[ext]
```

**Examples:**
- `fb-ad-spring-sale-v1-20260205.png`
- `yt-thumbnail-tutorial-ep5-v2-20260205.jpg`
- `li-carousel-case-study-card3-20260205.png`

### Platform Codes
- `fb` - Facebook
- `ig` - Instagram
- `li` - LinkedIn
- `yt` - YouTube
- `gd` - Google Display
- `tw` - Twitter/X

### Type Codes
- `ad` - Advertisement
- `thumbnail` - Video thumbnail
- `banner` - Cover/banner
- `post` - Social post
- `carousel` - Carousel card
- `story` - Story/Reel

## Quality Standards

### Must Have
- Correct dimensions for platform
- Brand colors accurate (hex values)
- Text readable at small sizes
- High resolution (no pixelation)
- Proper file format

### Should Have
- Mobile-first design
- Strong visual hierarchy
- Clear call-to-action area
- A/B test variations

### Nice to Have
- Multiple aspect ratios
- Animated versions
- Dark mode variants
